# AI-агент для анализа репозиториев и планирования задач

Трек: A+D

Это консольный Python-агент, который отвечает не только текстом, но и через инструменты: ходит в GitHub API и помогает выбрать лучший набор задач при ограниченном времени.

Главные сценарии:

- оценить публичный GitHub-репозиторий: активность, языки, issues, риски;
- распределить часы между задачами и сравнить простой план с оптимальным;
- собрать данные о репозитории и на их основе предложить план демо.

## Что умеет

**GitHub API (`github_repo_info`)**

Инструмент получает описание репозитория, звезды, форки, открытые issues, даты активности, лицензию и языки. Для публичных репозиториев токен не обязателен, но `GITHUB_TOKEN` увеличивает лимит запросов.

**Оптимизация плана (`plan_optimizer`)**

Инструмент принимает задачи с длительностью и ценностью. Сначала строит базовый жадный план по `value_per_hour`, затем считает улучшенный вариант как задачу о рюкзаке 0/1 через динамическое программирование.

## Установка

```bash
git clone https://github.com/andreysk0304/mat_mod_mai_2026_agent
cd mat_mod_mai_2026_agent
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -e .
```

Создайте `.env` из шаблона:

```bash
cp .env.example .env
```

Минимальные настройки:

```env
AI_AGENT_API_KEY=...
AI_AGENT_MODEL=gpt-4o-mini
AI_AGENT_BASE_URL=https://api.openai.com/v1
```

Опционально для GitHub:

```env
GITHUB_TOKEN=...
```

## Примеры для демо

### 1. Анализ репозитория

```text
Проанализируй репозиторий openai/openai-python: насколько он активен, какие языки использует и какие риски видны по открытым issues?
```

Ожидаемый инструмент:

```text
[tools: github_repo_info]
```

### 2. План на ограниченное время

```text
У меня есть 8 часов на мини-хак. Задачи:
- README 1ч, ценность 3
- инструмент GitHub API 4ч, ценность 10
- инструмент оптимизации 4ч, ценность 10
- REFLECTION 1ч, ценность 3
- скринкаст 3ч, ценность 8
- полировка консольного интерфейса 1ч, ценность 3
- тесты 3ч, ценность 6

Составь план, сравни базовый и улучшенный варианты, объясни что не делать.
```

Ожидаемый инструмент:

```text
[tools: plan_optimizer]
```

### 3. API + оптимизация

```text
Посмотри GitHub-репозиторий pallets/flask и предложи план из 8 часов для подготовки учебного демо: что показать, что проверить и что отложить. Сравни простой базовый вариант и оптимизированный план.
```

В этом сценарии агент обычно вызывает оба инструмента: сначала `github_repo_info`, затем `plan_optimizer`.

## Команды в CLI

- `/help` - помощь;
- `/tools` - список инструментов;
- `/reset` - очистить историю;
- `/reload` - перезагрузить инструменты после изменений;
- `/log on` и `/log off` - включить или выключить логирование;
- `/exit` - выйти.

После ответа CLI показывает, какие инструменты реально использовались:

```text
[tools: github_repo_info, plan_optimizer]
```

